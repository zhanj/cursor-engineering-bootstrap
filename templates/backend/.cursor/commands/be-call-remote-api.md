# /be-call-remote-api（backend）

目标：基于远端 OpenAPI 快速生成/补齐调用端，确保映射与异常处理可维护。

## 输入
- 远端 `openapi.yaml` 或接口文档
- 本端领域对象与错误码约定
- 超时/重试/鉴权策略

## 执行步骤
1. 生成或补齐 client 接口（按现有技术栈：Feign/WebClient/HttpClient）。
2. 建立 DTO 与领域对象映射，不泄漏第三方字段到核心域模型。
3. 统一异常映射（超时、4xx、5xx）到本端错误模型。
4. 增加最小测试（成功路径 + 失败路径至少各 1 条）。

## 输出格式（必须）
- `新增/修改文件`
- `映射策略`
- `异常策略`
- `验证命令`

## 禁止事项
- 直接在业务层散落 HTTP 调用。
- 忽略超时、重试、熔断等基础可靠性设置。
