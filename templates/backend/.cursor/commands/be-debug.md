# /be-debug（backend 四段式）

目标：定位并修复后端问题，输出可复现、可验证、可回归的结论。

## 四段式
1. `现象`：报错信息、触发条件、影响范围。
2. `定位`：最小复现路径 + 可疑模块（controller/service/dao/db）。
3. `修复`：最小补丁与原因。
4. `验证`：复现用例通过、回归命令通过。

## 必要输入
- 错误日志/堆栈
- 请求参数/调用链信息
- 最近相关改动（commit 或文件）

## 输出格式（必须）
- 先输出 debug 四段结论：
  - `Root Cause`
  - `Fix`
  - `Validation`
  - `Regression Risk`
- 再附加可直接粘贴到 `PR_TEMPLATE.md` 的字段：
  - **4) 验证证据（必填）**：补充 `本地验证命令` 与 `关键结果摘要`
  - **5) 风险与回滚（必填）**：补充 `主要风险` 与 `回滚方案`

## 约束
- 优先修复根因，不以吞异常/注释代码掩盖问题。
- 若涉及接口行为变化，必须同步契约或在输出中解释。
