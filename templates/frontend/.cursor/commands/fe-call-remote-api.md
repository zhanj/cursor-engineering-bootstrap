# /fe-call-remote-api（frontend）

目标：根据 OpenAPI 生成或补齐 `src/api`，统一请求与类型定义。

## 输入
- 远端 `openapi.yaml`
- 现有请求封装（拦截器、鉴权、错误处理）
- 前端代码风格（TS 类型、返回结构）

## 执行步骤
1. 在 `src/api` 生成/更新接口方法与类型定义。
2. 对齐统一请求封装（baseURL、token、错误码处理）。
3. 页面层仅调用 API 方法，不拼接 URL 字符串。
4. 至少给出 1 个调用示例（成功态 + 异常态处理）。

## 输出格式（必须）
- `新增/修改文件`
- `类型与返回模型策略`
- `错误处理策略`
- `验证命令`
