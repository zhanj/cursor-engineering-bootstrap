# /fe-debug（frontend 四段式）

目标：快速定位前端问题并提供可回归修复。

## 四段式
1. `现象`：页面位置、用户操作、报错信息。
2. `定位`：组件树、请求链路、状态流（router/store/api）。
3. `修复`：最小补丁与根因说明。
4. `验证`：复现路径通过 + lint/tsc 通过。

## 必要输入
- 控制台报错/网络请求信息
- 复现步骤
- 最近改动文件

## 输出格式（必须）
- 先输出 debug 四段结论：
  - `Root Cause`
  - `Fix`
  - `Validation`
  - `Regression Risk`
- 再附加可直接粘贴到 `PR_TEMPLATE.md` 的字段：
  - **4) 验证证据（必填）**：补充 `本地验证命令`、`页面验证清单`
  - **5) 风险与回滚（必填）**：补充 `主要风险` 与 `回滚方案`
